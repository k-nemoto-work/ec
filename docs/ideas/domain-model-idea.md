# ECサイト ドメインモデル アイデアメモ

> **前提**: リユース品（1点もの）を扱うECサイト。同一商品の複数在庫は存在しない。

## スコープ

- 商品カタログ（商品一覧・詳細・カテゴリ管理）
- カート（追加・削除）
- 注文管理（注文確定・注文履歴・ステータス管理）
- 顧客・認証（会員登録・ログイン・配送先住所管理）
- 決済・配送: ドメインオブジェクトとして表現するが **モック扱い**（外部連携なし）

---

## ドメインオブジェクト候補と集約

### 顧客集約

**ルート: 顧客 (Customer)**
- 顧客ID
- 名前
- メールアドレス
- パスワードハッシュ
- ステータス（活性 / 非活性）

**住所 (Address) ※値オブジェクト**
- 郵便番号
- 都道府県
- 市区町村
- 番地

### 商品集約

**ルート: 商品 (Product)**
- 商品ID
- 商品名
- 価格
- 商品説明
- ステータス（販売中 / 売約済み / 売却済み / 非公開）

**カテゴリ (Category)**
- カテゴリID
- カテゴリ名

※カテゴリを商品集約に含めるか独立した集約にするかは要検討

### カート集約

**ルート: カート (Cart)**
- カートID
- 顧客ID

**カートアイテム (CartItem)**
- 商品ID

### お気に入り集約

**ルート: お気に入りリスト (Favorite)**
- 顧客ID（1顧客につき1リスト）

**お気に入りアイテム (FavoriteItem)**
- 商品ID
- 追加日時

### 注文集約

**ルート: 注文 (Order)**
- 注文ID
- 顧客ID
- 注文日時
- 注文ステータス（未確定 / 確定 / 配送中 / 配送完了 / キャンセル）
- 合計金額

**注文明細 (OrderItem)**
- 商品名（スナップショット）
- 単価（スナップショット）

**決済 (Payment) ※モック**
- 決済方法
- 決済ステータス（未決済 / 決済済み）

**配送 (Shipment) ※モック**
- 配送先住所
- 配送ステータス（未発送 / 発送済み / 配送完了）

## ビジネスルール候補

### 顧客
- メールアドレスはユニーク
- 活性ステータスの顧客のみ注文可能
- パスワードは一定の強度を持つ必要がある

### 商品
- 価格は 0 より大きい値でなければならない
- 販売中ステータスの商品のみカートに追加できる

### カート
- 同じ商品の重複追加不可
- 売約済み・売却済みの商品はカートに追加できない
- 顧客 1 人につきカートは 1 つ

### お気に入り
- 同じ商品の重複追加不可
- 売却済み・非公開の商品はお気に入りに追加できない（販売中・売約済みは可）
- お気に入りに追加できる上限数（要検討）
- 顧客 1 人につきお気に入りリストは 1 つ

### 注文
- 注文確定時に商品ステータスを「売約済み」に変更する（既に売約済みの場合はエラー）
- 注文確定後は内容を変更できない
- 配送中・配送完了後はキャンセルできない
- 決済済みにならないと配送ステータスを進められない
- 注文明細には注文時点の商品名・価格をスナップショットとして保持する（後から商品情報が変わっても注文内容に影響しない）

## 境界付けられたコンテキスト（検討中）

まだ決定していないが、以下のような分割が考えられる:

| コンテキスト案 | 含まれるオブジェクト |
|---|---|
| カタログコンテキスト | 商品、カテゴリ |
| 注文コンテキスト | カート、注文、注文明細 |
| 顧客コンテキスト | 顧客、住所、お気に入り |
| 決済コンテキスト | 決済（モック） |
| 配送コンテキスト | 配送（モック） |

→ ドメインモデル図を作成してから、自然な分割を見直す予定
