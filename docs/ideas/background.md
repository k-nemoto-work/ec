# 🚀 ECサイト開発 技術スタック (2025年版)

ドメイン駆動設計 (DDD) の実践を目的とした、モダンで堅牢なECサイトを構築するための技術選定です。

---

### **バックエンド (Backend)**

> 
> 
> 
> ### **言語: ☕️ Kotlin**
> 
> - **選定理由**: Javaの強力なエコシステムを継承しつつ、より簡潔で安全なコード記述が可能。特に、`data class`や`sealed class`はDDDにおける値オブジェクトやドメインイベントの表現に最適。

> 
> 
> 
> ### **フレームワーク: 🌱 Spring Boot 3**
> 
> - **選定理由**: Kotlin/Javaにおけるデファクトスタンダード。豊富な機能と安定性を持ち、認証、DB連携、API開発まで網羅的にカバーできるため、ドメインロジックの実装に集中できる。

> 
> 
> 
> ### **アーキテクチャ: 🏛️ クリーンアーキテクチャ**
> 
> - **選定理由**: ビジネスロジック（ドメイン）をシステムの中心に据える設計思想。関心事の分離を徹底し、テスト容易性・保守性を最大化する。DDDとの親和性が極めて高い。

---

### **データ永続化 (Data Persistence)**

> 
> 
> 
> ### **データベース: PostgreSQL**
> 
> - **選定理由**: 仕事でMySQLを使っているので別のものを使ってみたかった。

> 
> 
> 
> ### **ORM: 🧩 Exposed**
> 
> - **選定理由**: JetBrains製のKotlinネイティブなSQLフレームワーク。型安全なDSLにより、SQLをKotlinコードで直感的に記述できる。ドメインモデルと永続化モデルの分離を自然に促し、JPAの課題であるインピーダンスミスマッチを回避しやすい。

> 
> 
> 
> ### **DBマイグレーション: ✈️ Flyway**
> 
> - **選定理由**: DBスキーマの変更履歴をバージョン管理するためのツール。SQLファイルベースでシンプルに運用でき、開発環境の再現性を担保する。

---

### **API & セキュリティ (API & Security)**

> 
> 
> 
> ### **API設計: 📡 REST**
> 
> - **選定理由**: Web APIの標準的な設計スタイル。HTTPメソッドとステータスコードに基づいた直感的なインターフェースを構築できる。ユースケース駆動でのエンドポイント設計と相性が良い。

> 
> 
> 
> ### **認証・認可: 🔐 Spring Security + JWT**
> 
> - **選定理由**: Springエコシステムにおける認証・認可の標準ライブラリ。JWT (JSON Web Token) と組み合わせることで、ステートレスでスケーラブルなAPIセキュリティを実現できる。

---

### **品質保証 & 運用 (Quality & Operations)**

> 
> 
> 
> ### **テスト: 🧪 JUnit + TestContainers**
> 
> - **選定理由**: ドメイン層のロジックを検証するユニットテストから、TestContainersを利用したDB連携テストまで、一貫したテスト戦略を可能にする。本番に近い環境でのテストにより、品質を大幅に向上させる。

> 
> 
> 
> ### **ドキュメント: 📖 OpenAPI (Swagger)**
> 
> - **選定理由**: コードからAPIドキュメントを自動生成する。フロントエンド開発者との円滑な連携を促進し、API仕様の「信頼できる唯一の情報源」となる。

> 
> 
> 
> ### **CI/CD: 🤖 GitHub Actions**
> 
> - **選定理由**: GitHubに統合されたCI/CDツール。リポジトリへのプッシュをトリガーに、テスト、ビルド、デプロイといったパイプラインを自動化できる。個人開発でも無料で始められる。

---

### **インフラ (Infrastructure)**

> 
> 
> 
> ### **環境構築: 🐳 Docker**
> 
> - **選定理由**: アプリケーションと実行環境をコンテナとしてパッケージングする技術。ローカル、ステージング、本番環境の差異をなくし、「自分の環境では動いたのに」という問題を解決する。
