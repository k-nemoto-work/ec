# 要求内容

## 概要

カートに追加した商品を注文確定し、注文履歴の参照・キャンセル・決済/配送ステータス更新を行う注文管理機能を実装する。

## 背景

カート機能が完成したため、次ステップとしてカートの商品を実際に注文確定するフローが必要。ECサイトのコアユースケースとして注文・決済・配送の状態遷移を管理する。

## 実装対象の機能

### 1. 注文確定（カートから注文作成）
- カート内の全商品を一括で注文確定する
- 注文時に商品名・価格のスナップショットを記録する
- 注文確定後にカートをクリアし、商品ステータスを `RESERVED` に変更する

### 2. 注文履歴一覧取得
- 自分の注文一覧をページネーション付きで取得できる
- 新しい注文順（`orderedAt` 降順）で返す

### 3. 注文詳細取得
- 指定した注文の詳細（アイテム・決済・配送情報含む）を取得できる
- 他人の注文へのアクセスは403エラー

### 4. 注文キャンセル
- `PENDING` または `CONFIRMED` 状態の注文のみキャンセル可能
- キャンセル時に商品ステータスを `ON_SALE` に戻す
- `SHIPPING` / `DELIVERED` 後はキャンセル不可

### 5. 決済ステータス更新（モック）
- 決済ステータスを `UNPAID` → `PAID` に更新する（外部連携なし）

### 6. 配送ステータス更新（モック）
- 配送ステータスを進める（`NOT_SHIPPED` → `SHIPPED` → `DELIVERED`）
- 決済ステータスが `PAID` でないと `SHIPPED` に進められない
- `DELIVERED` 時に商品ステータスを `SOLD` に変更

## 受け入れ条件

### 注文確定
- [ ] カートが空の場合は `BusinessRuleViolationException` が発生する
- [ ] カート内の商品が `ON_SALE` 以外の場合はエラー（`BusinessRuleViolationException`）
- [ ] 注文確定後にカートアイテムがクリアされる
- [ ] 注文確定後に商品ステータスが `RESERVED` になる
- [ ] 注文時に商品名・価格スナップショットが記録される
- [ ] 合計金額が正しく計算される

### 注文履歴・詳細
- [ ] 自分の注文のみ取得できる
- [ ] 他人の注文詳細取得で403が返る

### キャンセル
- [ ] `SHIPPING` 以降の注文はキャンセルできない（`BusinessRuleViolationException`）
- [ ] キャンセル後に商品ステータスが `ON_SALE` に戻る

### 決済・配送
- [ ] 未決済状態で配送ステータスを `SHIPPED` に進めようとするとエラー
- [ ] `DELIVERED` 時に商品ステータスが `SOLD` になる

## スコープ外

以下はこのフェーズでは実装しません:

- 実際の決済API連携
- 実際の配送業者API連携
- 管理者向け注文管理画面
